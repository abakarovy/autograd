# АвтоГрад — Автосалон

**Дипломный проект:** Проектирование и разработка базы данных для автосалона АвтоГрад

## Технологии

| Компонент      | Технология                      |
|----------------|---------------------------------|
| Frontend       | HTML5, CSS3, Vanilla JavaScript |
| Backend        | Python 3.10+, FastAPI           |
| База данных    | SQLite                          |
| ORM            | SQLAlchemy                      |
| Валидация      | Pydantic                        |
| Архитектура    | REST API                        |
| Документация   | Swagger UI (автогенерация)      |

## Запуск проекта

### 1. Создание виртуального окружения

```bash
cd autograd
python -m venv venv
```

Активация:

- **Windows (PowerShell):** `.\venv\Scripts\Activate.ps1`
- **Windows (CMD):** `venv\Scripts\activate.bat`
- **Linux/macOS:** `source venv/bin/activate`

### 2. Установка зависимостей

```bash
pip install -r backend/requirements.txt
```

### 3. Запуск сервера FastAPI

```bash
cd backend
uvicorn main:app --reload --host 127.0.0.1 --port 8000
```

### 4. Открытие сайта

- **Главная страница:** откройте `frontend/index.html` в браузере
- **Каталог:** `frontend/catalog.html`
- **Админ-панель:** `frontend/admin.html`
- **Swagger API:** [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

> При первом запуске база данных автоматически заполняется демонстрационными данными (12 автомобилей).

## API Endpoints

### Автомобили (`/cars`)

| Метод  | Путь             | Описание                              |
|--------|------------------|---------------------------------------|
| GET    | `/cars`          | Список с фильтрацией и пагинацией    |
| GET    | `/cars/brands`   | Список уникальных марок              |
| GET    | `/cars/{id}`     | Получить по ID                        |
| POST   | `/cars`          | Добавить автомобиль                   |
| PUT    | `/cars/{id}`     | Обновить автомобиль                   |
| DELETE | `/cars/{id}`     | Удалить автомобиль                    |

**Параметры фильтрации:** `search`, `brand`, `min_price`, `max_price`, `year`, `sort`, `page`, `per_page`

### Клиенты (`/clients`)

| Метод  | Путь              | Описание            |
|--------|--------------------|---------------------|
| GET    | `/clients`         | Список клиентов     |
| GET    | `/clients/{id}`    | Получить по ID      |
| POST   | `/clients`         | Добавить клиента    |
| PUT    | `/clients/{id}`    | Обновить клиента    |
| DELETE | `/clients/{id}`    | Удалить клиента     |

### Продажи (`/sales`)

| Метод  | Путь           | Описание                                       |
|--------|----------------|-------------------------------------------------|
| GET    | `/sales`       | Список продаж                                  |
| GET    | `/sales/{id}`  | Получить по ID                                  |
| POST   | `/sales`       | Оформить продажу (авто → available = false)     |

## База данных

### ER-диаграмма

```
┌──────────────┐       ┌──────────────┐       ┌──────────────┐
│     cars     │       │    sales     │       │   clients    │
├──────────────┤       ├──────────────┤       ├──────────────┤
│ id (PK)      │──┐    │ id (PK)      │   ┌──│ id (PK)      │
│ brand        │  │    │ car_id (FK)  │───┘  │ first_name   │
│ model        │  └────│ client_id(FK)│      │ last_name    │
│ year         │       │ sale_date    │      │ phone        │
│ price        │       │ final_price  │      │ email        │
│ mileage      │       └──────────────┘      │ passport_num │
│ fuel_type    │                              └──────────────┘
│ transmission │
│ body_type    │
│ color        │
│ engine_volume│
│ description  │
│ image_url    │
│ available    │
└──────────────┘
```

**Связи:**
- `cars (1) → (N) sales` — один автомобиль может иметь запись о продаже
- `clients (1) → (N) sales` — один клиент может совершить несколько покупок
- Каскадное удаление: при удалении автомобиля или клиента удаляются связанные записи продаж

### Нормализация (3NF)

- **1NF:** Все атрибуты атомарны, каждая таблица имеет первичный ключ
- **2NF:** Нет частичных зависимостей (все неключевые атрибуты зависят от всего ключа)
- **3NF:** Нет транзитивных зависимостей между неключевыми атрибутами

## Возможности системы
